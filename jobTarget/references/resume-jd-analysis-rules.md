# Resume and JD Analysis Rules

Use this file after the first-pass script output.

## Match categories

### Direct match

- same function, scope, or toolset
- clear evidence in projects or achievements
- candidate can defend the depth in an interview

### Transferable match

- adjacent domain or similar stakeholder pattern
- same problem type solved in a different setting
- enough proof to build a convincing story

### Weak match

- buzzword overlap without demonstrated ownership
- tool familiarity without results
- academic or side-project exposure only

## Gap categories

### Hard gap

- required domain, stack, or level is missing
- interviewer is likely to test it directly

### Soft gap

- candidate has adjacent experience but needs better framing

### Cosmetic gap

- JD wording differs, but the underlying capability exists

## Prep prioritization

Rank prep items by:

1. interview likelihood
2. business criticality
3. ability to close quickly
4. downside if exposed

## What to output

For each major gap, provide:

- why it matters
- how severe it is
- how to prepare
- what story or proof the candidate should use

## Avoid these mistakes

- Do not treat keyword overlap as proof of fit.
- Do not punish candidates for wording differences if the delivered outcomes are similar.
- Do not over-credit side projects when the role requires production ownership.
